# Backend API documentation

This folder contains the API contract and domain flow descriptions for the Boonma Farm backend.

## Contents

| Artifact                            | Description                                                                                                                           |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **openapi.yaml**                    | OpenAPI 3.0 specification for all implemented REST APIs under `/api/v1`. Use for client generation, validation, or MCP tooling.       |
| **flows/**                          | Domain flow documents (one markdown file per area) describing purpose, actors, endpoints, and errors.                                 |
| **swagger.yaml** / **swagger.json** | Swagger 2.0 spec generated from code. Served at `GET /swagger/*` (Swagger UI). Regenerate with `make gen-swag` from the backend root. |
| **docs.go**                         | Go package generated by swag; used by Swagger UI. Do not edit by hand.                                                                |

## Flow documents

- [flows/auth.md](flows/auth.md) – Register, login, logout; JWT and cookie.
- [flows/client.md](flows/client.md) – Client CRUD; super admin vs client-scoped access.
- [flows/user.md](flows/user.md) – User CRUD and list; client-scoped.
- [flows/farm-hierarchy.md](flows/farm-hierarchy.md) – Farm CRUD, list, and hierarchy with ponds.
- [flows/pond.md](flows/pond.md) – Pond CRUD; create many by farm; list by farmId.
- [flows/pond-stock-actions.md](flows/pond-stock-actions.md) – Pond stock actions overview (fill, move, sell) and active pond lifecycle.
- [flows/pond-stock-fill.md](flows/pond-stock-fill.md) – Fill (add fish); creates active pond if pond is in maintenance.
- [flows/pond-stock-move.md](flows/pond-stock-move.md) – Move (transfer fish); destination pond may become active.
- [flows/pond-stock-sell.md](flows/pond-stock-sell.md) – Sell; optional markToClose to return pond to maintenance.
- [flows/worker.md](flows/worker.md) – Worker CRUD and list; client-scoped.
- [flows/merchant.md](flows/merchant.md) – Merchant CRUD and list; global list; only super admin can add.
- [flows/feed-collection.md](flows/feed-collection.md) – Feed collection CRUD; pagination and keyword search.
- [flows/feed-price-history.md](flows/feed-price-history.md) – Feed price history CRUD and list.

## Quick reference

- **Base URL**: `http://localhost:8080` (or your server)
- **API prefix**: `/api/v1`
- **Auth**: JWT in `Authorization: Bearer <token>` or in cookie `jwt_token`. Auth endpoints (`/api/v1/auth/*`) do not require a token.
- **Success response**: `{ "result": true, "data": ... }`
- **Error response**: `{ "code": "<string>", "message": "<string>" }`
